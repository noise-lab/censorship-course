# Class Meeting Agenda Template (3-hour summer)

This is the summer-term agenda template for 3-hour class meetings. For the
80-minute standard-term format, see [agenda-template.md](agenda-template.md).

Each 3-hour class meeting is built around the same four blocks as the
standard-term format — **lecture**, **reading response discussion**,
**student presentation**, and **hands-on activity** — plus a dedicated
**breakout discussion** block. Two 10-minute breaks are built in.

Two motifs work well. Pick whichever fits the day's material. For
Lecture 1, when there is no assigned reading and no student presentation
scheduled, see the [Lecture 1 variant](#lecture-1-variant-no-reading-no-student-presentation) at the bottom.

## Motif A — Discussion sandwich

Best when the assigned reading is dense and student-led discussion should
carry the room's energy into the lecture.

| Time      | Min | Block                                        | Notes                                |
|-----------|-----|----------------------------------------------|--------------------------------------|
| 0:00–0:10 |  10 | Housekeeping + warm-up prompt                |                                      |
| 0:10–0:35 |  25 | Reading response Q&A                         | Instructor facilitates               |
| 0:35–0:55 |  20 | Student presentation                         | Student-led                          |
| 0:55–1:05 |  10 | Break                                        |                                      |
| 1:05–1:50 |  45 | Lecture                                      | Book concepts and technical framing  |
| 1:50–2:20 |  30 | Hands-on activity                            | Groups or pairs                      |
| 2:20–2:30 |  10 | Break                                        |                                      |
| 2:30–2:55 |  25 | Breakout discussion + report-back            | Groups pick one of two topics        |
| 2:55–3:00 |   5 | Wrap + reading preview                       |                                      |

## Motif B — Frontloaded lecture

Best when the day introduces new technical machinery students need before
they can meaningfully discuss. Puts the biggest student-led block at the end.

| Time      | Min | Block                                        | Notes                                |
|-----------|-----|----------------------------------------------|--------------------------------------|
| 0:00–0:10 |  10 | Housekeeping + reading Q&A quick hits        |                                      |
| 0:10–0:55 |  45 | Lecture                                      | Book concepts and technical framing  |
| 0:55–1:05 |  10 | Break                                        |                                      |
| 1:05–1:30 |  25 | Student presentation                         | Student-led                          |
| 1:30–2:15 |  45 | Hands-on activity                            | Groups or pairs                      |
| 2:15–2:25 |  10 | Break                                        |                                      |
| 2:25–2:55 |  30 | Breakout discussion + report-back            | Groups pick one of two topics        |
| 2:55–3:00 |   5 | Wrap + reading preview                       |                                      |

## Lecture 1 variant (no reading, no student presentation)

The first meeting has no assigned reading and no student presentation
scheduled. Substitute the reading Q&A + student-presentation slots with
introductions and a syllabus tour.

| Time      | Min | Block                                        | Notes                                        |
|-----------|-----|----------------------------------------------|----------------------------------------------|
| 0:00–0:15 |  15 | Introductions + expectations                 |                                              |
| 0:15–1:00 |  45 | Lecture 1: Information Control framework     | Book Chapter 1                               |
| 1:00–1:10 |  10 | Break                                        |                                              |
| 1:10–1:40 |  30 | Syllabus tour                                | Grading, AI policy, project, reading responses |
| 1:40–2:25 |  45 | Hands-on: [mapping the Internet control stack](activities/full-stack.md) |                                              |
| 2:25–2:35 |  10 | Break                                        |                                              |
| 2:35–2:55 |  20 | [Breakout discussion](breakouts/full-stack.md) + report-back | Pick A (age-verification) or B (consolidation) |
| 2:55–3:00 |   5 | Wrap + Lecture 2 reading preview             |                                              |

## Notes on adaptation

- **Reading Q&A.** Students post reading responses ahead of class; use the
  Q&A slot to work through the most common questions or points of confusion.
- **Student presentations.** ~20 minutes of material; leave room for
  audience Q&A after. See the [syllabus](syllabus.md#grading) for topic
  and format guidance.
- **Hands-on activities.** Weeks with a heavy lab (DNS, BGP, throttling
  detection, OONI) may need the full 45 minutes; lighter case-study
  activities can be 30 and reclaim time for breakout.
- **Breakouts.** See `docs/breakouts/<topic>.md` for two debate topics per
  lecture; groups pick one and report back.
- **Breaks.** Two 10-minute breaks work well for a 3-hour block. Move them
  earlier or later depending on which activity is running long.
